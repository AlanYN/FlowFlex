<objective>
Three independent frontend-only changes that improve Case page layout, Stage interactivity, and Short Answer Grid validation.

Purpose: UX-02 reduces header visual weight by moving the status tag inline with the Case name. UX-03 lets users collapse the Stage detail region to focus on stage-level navigation. DATA-02 relaxes an overly strict grid required validation so users are not blocked when they have partially filled a grid.

Output:
- detail.vue — status tag moved from #description slot into #title slot with flex wrapper (per D-01, D-02)
- EditableStageHeader.vue — isExpanded ref + el-collapse-transition wrapping the detail body, @click.stop on edit button (per D-03, D-04, D-05, D-06)
- dynamicForm.vue + QuestionnaireDetails.vue — short_answer_grid required check changed from every-row to any-cell (per D-10, D-11, D-12)
</objective>
