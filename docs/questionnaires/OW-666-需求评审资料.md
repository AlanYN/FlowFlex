# OW-666: Repeatable Section in Questionnaire — 需求评审资料

## 一、需求概述

### 背景

FlowFlex 的 Questionnaire（问卷）系统目前支持将问卷拆分为多个 Section（分区），每个 Section 包含若干 Question。Assignee（填写人）按 Section 分页依次填写。

**当前的问题是：** 每个 Section 只能填写一组答案。但实际业务中存在"行数不确定"的采集场景 —— 比如收集客户的仓库信息，有的客户 1 个仓库，有的 5 个，每个仓库都要填同一组问题（仓库名、负责人、地址……）。现在没法做到。

### 本票目标

为 Section 增加"可重复"（Repeatable）能力：
- **Admin 配置时**：可以把某个 Section 标记为 Repeatable
- **Assignee 填写时**：可以动态"加一组 / 删一组"相同问题集合

### 业务示例

**示例 1：按仓库分配团队**
```
Section: "Facility Configuration" (Repeatable = true)
  ├─ Facility Name (Text, Required)
  ├─ Account Manager (People, Required)
  ├─ Team Lead (People, Required)
  ├─ Operations Manager (People)
  └─ Warehouse Manager (People)

填写效果：
  Group 1: LA Warehouse | Alice | Bob | Carol | Dave
  Group 2: NJ Warehouse | Eve | Frank | Grace | Henry
  [+ Add Another Facility]
```

**示例 2：按运输线路配置服务**
```
Section: "Lane Details" (Repeatable = true)
  ├─ Lane Origin (Text, Required)
  ├─ Lane Destination (Text, Required)
  ├─ Service Type (Text)
  ├─ Weekly Volume (Number)
  └─ Go-Live Date (Date Picker)

填写效果：
  Group 1: Shanghai → LA | LCL | 50 TEU | 2026-08-01
  Group 2: Shenzhen → NJ | FCL | 30 TEU | 2026-08-15
  [+ Add Another Lane]
```

## 二、验收标准（共 11 条）

### Admin 配置端

| # | 需求 | 验收标准 |
|---|------|---------|
| 1 | Section 增加 Repeatable 开关 | 有问卷编辑权限的用户即可开启/关闭 Repeatable，默认关闭 |
| 2 | Repeatable Section 视觉标识 | 编辑页面中 Repeatable Section 有图标或标签区分于普通 Section |
| 3 | Question 配置方式不变 | 所有现有 Question Type（Single-line Text、People、Date Picker、Number、File 等）均可在 Repeatable Section 中使用 |

### Assignee 填写端

| # | 需求 | 验收标准 |
|---|------|---------|
| 4 | 默认展示 1 组空白 | 打开含 Repeatable Section 的问卷时，默认显示第 1 组 |
| 5 | "Add Another" 新增一组 | 点击后新增一组完整 Question 集合，组编号自动递增（Group 1, Group 2, Group 3…） |
| 6 | 删除某一组 | 每组有删除按钮，删除后编号自动重排；至少保留 1 组 |
| 7 | 每组独立校验 Required | 每一组的 Required Question 都必须填写，未填时阻止提交并定位到具体组和问题 |
| 8 | 提交保存所有组 | 提交时保存全部组的答案，Questionnaire Instance 标记为已提交 |

### 数据模型

| # | 需求 | 验收标准 |
|---|------|---------|
| 9 | 新增 Section Instance 概念 | 一个 Repeatable Section 模板对应 N 个 Section Instance，每个 Instance 含一组完整 Answer |
| 10 | 向后兼容 | 非 Repeatable Section 行为不变；已有问卷数据不受影响；无需数据迁移 |

### 数据导出

| # | 需求 | 验收标准 |
|---|------|---------|
| 11 | Export 支持 Repeatable 数据 | Repeatable Section 每组展平为一行导出 |

## 四、产品决策（已确认）

| 决策 | 说明 |
|------|------|
| 在途 Case 策略 | 模板修改仅影响新 Case，已提交的 Instance 不变 |
| 权限 | 无需额外 Admin 权限，有问卷编辑权限即可操作 Repeatable 开关 |
| 默认状态 | Repeatable 默认关闭，需手动开启 |

## 五、当前系统架构（技术现状）

### 5.1 核心概念说明

| 概念 | 说明 |
|------|------|
| **Questionnaire** | 问卷模板，由 Admin 创建和配置 |
| **Section** | 问卷中的分区，一份问卷可有多个 Section，Assignee 按 Section 分页填写 |
| **Question** | Section 中的具体问题，有多种类型（文本、数字、日期、人员、文件等） |
| **QuestionnaireAnswer** | 填写人提交的答案记录，关联到具体的 Onboarding Case + Stage |
| **Onboarding** | 一个客户入驻流程实例（Case），包含多个 Stage |
| **Stage** | 流程阶段，一个 Stage 可以挂载一份或多份 Questionnaire |

### 5.2 数据存储方式

**问卷模板（Section + Question 定义）：**
- 存储在 `ff_questionnaire` 表的 `structure_json` 字段（PostgreSQL JSONB）
- 没有独立的 Section 表（历史上有过 `ff_questionnaire_section` 表，已废弃删除）
- JSON 结构示意：

```json
{
  "sections": [
    {
      "id": "snowflake-id",
      "title": "Section Title",
      "description": "...",
      "order": 0,
      "questions": [
        {
          "id": "snowflake-id",
          "title": "Question text",
          "type": "short_answer | paragraph | multiple_choice | number | date | file | ...",
          "required": true,
          "options": [...],
          "jumpRules": [...]
        }
      ]
    }
  ]
}
```

**问卷答案：**
- 存储在 `ff_questionnaire_answers` 表
- 一条记录 = 一次填写（通过 `onboarding_id` + `stage_id` + `questionnaire_id` 唯一定位）
- 答案内容存在 `answer_json` 字段（JSONB），结构为：

```json
{
  "responses": [
    {
      "questionId": "问题ID",
      "question": "问题文本",
      "answer": "用户填写的值",
      "type": "question 类型",
      "responseText": "补充文本（如 Other 选项的自定义内容）"
    }
  ]
}
```

- 支持版本管理：`version` + `is_latest` 字段
- 状态追踪：`status`（Draft / Submitted / Approved）、`completion_rate`（0-100）

**关键特征：没有"Section Instance"概念** —— 当前答案是一个扁平的 responses 数组，不按 Section 分组。

### 5.3 前端架构

| 场景 | 组件 | 说明 |
|------|------|------|
| Admin 编辑 Section | `SectionManager.vue` | 左侧面板，管理 Section 列表（拖拽排序、增删） |
| Admin 编辑 Question | `QuestionEditor.vue` | 右侧区域，编辑具体 Question 配置 |
| Assignee 填写 | `dynamicForm.vue` | 按 Section 分页渲染，支持 Previous/Next 导航和进度点 |
| 数据导出 | `customer-overview.vue` | 纯前端 Excel 导出（SheetJS），展平为每行一个 Question |

### 5.4 前端 Section 类型定义

```typescript
// packages/flowFlex-common/src/types/section.d.ts
export interface Section {
    id?: string;
    temporaryId: string;
    name: string;
    description?: string;
    questions: QuestionnaireSection[];
    order?: number;
    isDefault?: boolean;
}
```

当前没有 `isRepeatable` 字段，需要本次新增。

## 七、数据结构变更方案

### 7.1 问卷模板 structure_json 的改动

Section 对象新增 `isRepeatable` 字段，改动极小：

```json
{
  "sections": [
    {
      "id": "s1",
      "title": "Company Information",
      "isRepeatable": false,
      "questions": [...]
    },
    {
      "id": "s2",
      "title": "Facility Configuration",
      "isRepeatable": true,
      "questions": [...]
    }
  ]
}
```

### 7.2 答案 answer_json 的改动

**当前结构（完全扁平，无 Section 分组）：**

```json
{
  "responses": [
    { "questionId": "q1", "question": "Company name?", "answer": "Acme", "type": "short_answer" },
    { "questionId": "q2", "question": "Business desc?", "answer": "Logistics", "type": "paragraph" }
  ]
}
```

**引入 Repeatable 后的核心问题：** 同一个 questionId 会出现 N 次答案（每组一次），当前的扁平结构无法区分哪个答案属于哪个 Group。

**推荐方案 —— 新增 `sectionInstances` 字段（物理隔离）：**

```json
{
  "responses": [
    // 非 Repeatable Section 的答案 —— 保持不变
    { "questionId": "q1", "question": "Company name?", "answer": "Acme", "type": "short_answer" }
  ],
  "sectionInstances": [
    // Repeatable Section 的答案 —— 按组存储
    {
      "sectionId": "s2",
      "groupIndex": 0,
      "responses": [
        { "questionId": "q10", "question": "Facility Name", "answer": "LA Warehouse", "type": "short_answer" },
        { "questionId": "q11", "question": "Account Manager", "answer": "Alice", "type": "people" }
      ]
    },
    {
      "sectionId": "s2",
      "groupIndex": 1,
      "responses": [
        { "questionId": "q10", "question": "Facility Name", "answer": "NJ Warehouse", "type": "short_answer" },
        { "questionId": "q11", "question": "Account Manager", "answer": "Eve", "type": "people" }
      ]
    }
  ]
}
```

**选择此方案的理由：**

1. **向后兼容** —— 非 Repeatable 答案留在 `responses` 不动，旧代码/旧数据零影响
2. **物理隔离** —— 旧代码不认识 `sectionInstances` 字段就自动忽略，不会出错
3. **导出友好** —— 遍历 `sectionInstances` 数组天然对应"每组一行"的导出要求
4. **Action/Condition 安全** —— 见下一节分析

### 7.3 对 Action / Condition Rule 的影响分析

**背景：** FlowFlex 有一套 Stage Condition Rules Engine —— 当阶段完成时，系统根据问卷答案值评估条件表达式，触发 GoToStage、SkipStage、TriggerAction 等动作。

**当前的读取路径：**

```
Stage 完成 → RulesEngineService.EvaluateConditionAsync()
  → ComponentDataService.GetQuestionnaireDataAsync()
    → 遍历 answer_json.responses[]
    → 构建 answerDict[questionId] = value （一对一映射）
  → RulesEngine 求值表达式：
    input.questionnaire.answers["questionnaireId"]["questionId"] == "someValue"
```

**核心假设：一个 questionId 只有一个答案值。** 整个 Rules Engine、ActionContextBuilder 都建立在这个假设上。

**Repeatable Section 的冲突：** 同一个 questionId 有 N 个值（每组一个），如果混入 `responses` 数组，`answerDict[questionId] = value` 会互相覆盖（只保留最后一组），产生不可预测的 bug。

**推荐方案为何安全：**

| 消费方 | 行为 | 安全性 |
|--------|------|--------|
| `ComponentDataService` | 只遍历 `responses` 数组 | 看不到 `sectionInstances`，不受影响 |
| `RuleInputDataBuilder` | 从 ComponentDataService 拿数据 | 间接安全 |
| `ActionContextBuilder` | 只读 `responses` 构建 answerByQuestionId | 间接安全 |
| 前端 Condition 配置 | fieldPath = `answers["qId"]["questionId"]` | Repeatable Section 的问题不会出现在可选列表中 |

**防护措施：** 前端配置 Condition Rule 时，属于 Repeatable Section 的 Question 应禁止作为条件字段被选择（灰显/不出现在下拉列表中）。

**MVP 已排除条件逻辑：** 票明确写了"条件逻辑 / 嵌套 Repeatable"不在 MVP 范围内，进一步确认了当前阶段不需要让 Rules Engine 读取 Repeatable 数据。

## 八、待评审确认事项

以下问题建议在评审中确认：

| # | 问题 | 建议 |
|---|------|------|
| 1 | 答案存储方案：扩展 `answer_json` 加 `sectionInstances` 字段 vs 新建数据库表？ | 建议扩展 JSON（保持架构一致性，不引入新表；物理隔离保证 Action/Condition 安全） |

| 5 | 是否需要 API 层面的 breaking change？ | 建议向后兼容，answer_json 扩展新字段，旧格式不变 |
| 6 | Repeatable Section 的 Question 是否需要禁止被 Condition Rule 引用？ | 建议禁止（MVP 不含条件逻辑，且技术上无法一对一取值） |
