## Phase 3：Assignee 填写端（核心）

### Task 3.1：sectionGroups 数据结构初始化
**改动文件：** `packages/flowFlex-common/src/app/views/onboard/onboardingList/components/dynamicForm.vue`

- 新增 `sectionGroups` reactive 对象：`Record<string, Record<string, any>[]>`
- 在 `onMounted` 中，对 `isRepeatable === true` 的 Section 初始化 1 组空白数据
- 非 Repeatable Section 继续使用现有 `formData`
- 验证：console.log 确认初始化结构正确

### Task 3.2：Repeatable Section 模板渲染
**改动文件：** `dynamicForm.vue`

- 在现有 Section 渲染逻辑外层增加 `v-if="!currentSection.isRepeatable"` / `v-else` 分支
- Repeatable 分支：`v-for="(group, groupIndex) in sectionGroups[currentSection.id]"`
  - 每组渲染 Group 标题："Group {{ groupIndex + 1 }}"
  - 每组渲染删除按钮（最后一组时隐藏）
  - 内部 question 渲染使用 `getTarget(groupIndex)[question.id]` 绑定
- 底部渲染 "Add Another Group" 按钮（组数 >= 50 时隐藏）
- 验证：页面能正确展示多组问题，输入数据互不干扰

### Task 3.3：增删 Group 逻辑
**改动文件：** `dynamicForm.vue`

- `addGroup(sectionId)`：向 `sectionGroups[sectionId]` push 一个空白 group 对象（按该 section 的 questions 初始化 key）
- `removeGroup(sectionId, groupIndex)`：有数据时弹确认框，确认后 splice 删除
- 验证：增删操作正确，编号自动重排，至少保留 1 组

### Task 3.4：getTarget 函数 + v-model 绑定
**改动文件：** `dynamicForm.vue`

- 实现 `getTarget(groupIndex?: number)` 函数
- Repeatable 模板中所有 `v-model` 替换为 `getTarget(groupIndex)[...]` 形式
- 包括 grid 类型的复合 key：`getTarget(groupIndex)[${question.id}_${row.id}]`
- 文件上传 `handleFileChange` 适配 group 维度
- 验证：各种 Question Type 在 Repeatable Section 中输入正确存储

### Task 3.5：Required 校验适配
**改动文件：** `dynamicForm.vue`

- `validateForm` 扩展：对 Repeatable Section 遍历所有 Group 逐组校验
- 错误格式包含 groupIndex：`{ sectionIndex, groupIndex, questionId }`
- 校验失败时跳转到对应 Section + 滚动到具体 Group 的具体 Question
- 验证：未填必填项时阻止提交，定位到正确的 Group 和 Question

### Task 3.6：transformFormDataForAPI 适配
**改动文件：** `dynamicForm.vue`

- 对 Repeatable Section：遍历 `sectionGroups[section.id]` 的每个 group，生成 `sectionInstances` 数组
- 对非 Repeatable Section：保持现有逻辑，放入 `responses`
- 最终输出：`{ responses: [...], sectionInstances: [...] }`
- 验证：提交后查看数据库 answer_json 结构正确

### Task 3.7：答案回填（Draft 恢复）
**改动文件：** `dynamicForm.vue`

- `applyAnswers` 扩展：读取 answer_json 中的 `sectionInstances` 字段
- 按 `sectionId` 匹配到对应 Repeatable Section
- 根据 group 数量动态创建 Groups，回填数据到 `sectionGroups`
- 验证：保存 Draft 后重新打开，多组数据完整恢复

### Task 3.8：completion_rate 计算适配
**改动文件：** `packages/flowFlex-common/src/app/views/onboard/onboardingList/components/QuestionnaireDetails.vue`

- `completionStats` computed 扩展：Repeatable Section 按 `实际组数 × 题目数` 计入 totalQuestions
- answeredQuestions 遍历 `sectionGroups` 每组每题
- 验证：增删组时进度率实时变化，数值正确
