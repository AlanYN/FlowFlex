# 原始需求存档

## 2026-05-11 — 初始需求

### 背景

WFE 的 Integration 模块已有字段映射功能：配置 WFE 字段 → 目标 API 字段的对应关系。但当目标 API 的字段是选项型（枚举、用户 ID、设施代码等）时，WFE 存的自由文本无法直接传递，导致 API 调用失败。

Amanda 设计了一套 AI 字段匹配方案（见 `amanda给的ai方案.md`），核心思路是在现有 field mapping 上加可选 `lookup` 属性，运行时实时拉选项列表 + 一次 LLM 调用完成匹配。

### 本阶段决策

由于 Amanda 尚未确认是否采用 AI 匹配方案，决定先开发"从第三方拉取选项数据"这一基础设施部分（不含 AI 匹配逻辑）。理由：

1. 不管最终是否用 AI，从第三方系统拉取选项列表是必须的
2. 即使用人工选择，审批人也需要从真实选项中选值
3. 基础设施先建好，等 AI 方案确认后只需加一层 FieldMatchingService

### 核心需求

- 在 Action 编辑弹窗的字段映射区域，支持为特定字段配置 lookup（选项来源）
- 运行时通过 Integration 的认证信息自动调用第三方 API 获取选项列表
- 提供预览功能验证配置正确性
- 单字段 lookup 失败不阻断整体流程
