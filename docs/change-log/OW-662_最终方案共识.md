> **参与人**: Kai Li, 领导

## 核心决策

### 同步方式：拉取模式（Pull）

**决策**：WFE 只暴露查询 API，Ticketing System 自行轮询拉取 ChangeLog。WFE 不主动调用任何外部系统。

**理由**：领导明确要求不能让调用外部接口的操作影响 Workflow 的正常使用。拉取模式下 WFE 完全被动，零风险。

**时效性**：分钟级延迟可接受。合同审批不是实时交易场景。

### 工作项 2：新增对外 ChangeLog 查询 API

**端点**：`GET /integration/external/v1/change-logs`

**查询参数**：

| 参数        | 类型              | 必填 | 说明                                         |
| ----------- | ----------------- | ---- | -------------------------------------------- |
| `entityId`  | string            | ✅   | 外部系统的实体 ID（如 Ticket ID "TK-12345"） |
| `since`     | ISO 8601 datetime | 否   | 增量拉取：只返回该时间之后的记录             |
| `pageIndex` | int               | 否   | 页码，默认 1                                 |
| `pageSize`  | int               | 否   | 页大小，默认 20                              |

**响应格式**：

```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": "1234567890123",
        "timestamp": "2026-07-02T10:15:00Z",
        "operator": {
          "id": "111",
          "name": "Amanda Li"
        },
        "operationType": "CaseUpdate",
        "title": "Case Updated: Contract - ABC Inc",
        "description": "Company Name changed from 'ABC Inc' to 'ABC Logistics'",
        "changes": [
          {
            "field": "Company Name",
            "oldValue": "ABC Inc",
            "newValue": "ABC Logistics"
          },
          {
            "field": "Priority",
            "oldValue": "Medium",
            "newValue": "High"
          }
        ]
      },
      {
        "id": "1234567890124",
        "timestamp": "2026-07-02T10:20:00Z",
        "operator": {
          "id": "222",
          "name": "John Wang"
        },
        "operationType": "StageTransition",
        "title": "Stage Moved: Draft → Legal Review",
        "description": "Case moved from stage 'Draft' to 'Legal Review'",
        "changes": [
          {
            "field": "Stage",
            "oldValue": "Draft",
            "newValue": "Legal Review"
          }
        ]
      }
    ],
    "totalCount": 42,
    "pageIndex": 1,
    "pageSize": 20
  }
}
```

**关键设计**：

- `changes` 数组由后端从 `before_data` / `after_data` JSON 中解析生成
- 每个 change 项包含 `field`（人可读字段名）、`oldValue`、`newValue`
- 对于非字段变更类型（如 StageTransition、TaskComplete），同样拆解为 `changes` 格式
- 按 `timestamp` 倒序排列

### 工作项 3：后端 diff 解析服务

新增一个 Service 方法，负责将内部的 `beforeData/afterData/changedFields` JSON 快照转换为扁平的 `changes` 数组。

逻辑类似前端 `parseStaticFieldChanges` 函数，但放在后端实现：

- 根据 `changedFields` 列表遍历
- 从 `beforeData` 取 oldValue，从 `afterData` 取 newValue
- 字段名映射为人可读名称（CaseName → "Customer Name"，Ownership → "Owner" 等）

## Ticketing System 集成流程

```
1. Ticketing 创建 Case（已有 API）
   POST /integration/external/v1/cases
   { systemId, workflowId, entityType: "ticket", entityId: "TK-12345", caseName, ... }
   → 返回 { caseId, caseCode }

2. WFE 用户操作 Case（正常使用，无变化）
   → ChangeLog 自动写入 ff_operation_change_log

3. Ticketing 轮询拉取 ChangeLog（新增 API）
   GET /integration/external/v1/change-logs?entityId=TK-12345&since=2026-07-02T10:00:00Z
   → 返回结构化的 changes 数组

4. Ticketing 展示为 Ticket Activity（Ticketing 侧实现，WFE 不关心）
```
